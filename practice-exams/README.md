## Taking the Exam

1. Set a 2-hour timer
2. Use only official Kubernetes documentation
3. No external resources or copying code
4. Read each question carefully
5. Focus on troubleshooting (30% of exam)

## Tips

- **Read Questions Carefully**: Don't miss important details
- **Focus on Troubleshooting**: 30% of exam is troubleshooting
- **Use kubectl Efficiently**: Learn common commands and aliases
- **Practice YAML**: Understand manifest structure
- **Time Management**: Don't spend too long on one question
